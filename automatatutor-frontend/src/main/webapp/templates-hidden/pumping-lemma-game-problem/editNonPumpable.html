Syntax - Unpumpable Languages

A language is considered unpumpable, if it is not regular and does not satisfy the pumping lemma. Check your language for these criteria before specifying them.
An arithmetic language consists of an alphabet, a flat symbolic string and constraints.

The alphabet is a set of letters ([a-zA-Z]) separated by spaces.

Let Σ be the alphabet and E be any set of linear combinations. A flat symbolic string is defined by the following BNF:
<P> ::= <P><P> | <I>e | σ | ε
<I> ::= <I><I> | σ | ε
where σ ∈ Σ and e ∈ E.

Let E be any set of linear combinations. The variables in the linear combinations must appear in the given symbolic string. The constraints of an arithmetic language defined by the following BNF:
<C> ::= <C> && <C> | e == e | e < e | e > e | e ≤ e | e ≥ e

Additionally, in case of an unpumpable language, you have to give an unpumpable word in that language depending on the pumping number n. The syntax ist the same as in a symbolic string, except that the only integer variable usable in linear combinations is n.

Syntax - Regular Languages

When defining a regular language with an arithmetic langauge, the symbolic string does not have to be flat. In this case, it is defined by the following BNF:
<P> ::= <P><P> | <P>e | σ | ε
where σ ∈ Σ and e ∈ E.

The constraints have to fulfil additional criteria: each variable may only appear once and each of the constraints must be unary (i.e. only contain exactly one variable).
